# CompliAI Diana / Partner Persona Lens

## Overview

Use this skill when reviewing partner onboarding, portfolio, client intake, client context, Resolve, Cockpit, Dosar, reporting, or alerts through Diana's eyes.

Diana wants:
- portfolio triage
- correct client context
- efficient execution
- clean deliverables
- no context loss

She does not want:
- solo-first flows disguised as partner
- cross-client ambiguity
- scattered output
- extra clicks without client value

## Workflow

1. Name Diana's stage:
   - awareness
   - auth
   - partner onboarding
   - portfolio
   - client intake
   - client workspace
   - client cockpit
   - client dossier
   - portfolio monitoring
2. Ask only these questions:
   - Does she know which client she is acting on?
   - Can she see what is urgent across clients?
   - Is execution client-safe and traceable?
   - Can she produce something credible for the client?
   - Is the product saving time or creating admin work?
3. Mark the surface as:
   - `scalabil pentru Diana`
   - `fragil pentru Diana`
   - `periculos cross-client`
4. If wrong, rewrite toward:
   - clearer client context
   - stronger portfolio triage
   - shorter route to case execution
   - cleaner client output

## Guardrails

- Do not collapse partner into solo.
- Do not allow loose cross-client execution.
- Do not make reporting an afterthought.
- Do not bury client identity while acting.
