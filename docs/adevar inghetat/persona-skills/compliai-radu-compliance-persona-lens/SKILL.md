# CompliAI Radu / Compliance Persona Lens

## Overview

Use this skill when reviewing compliance onboarding, Home, Resolve, Cockpit, Dosar, Monitoring, revalidation, export, audit, or specialist depth through Radu's eyes.

Radu wants:
- truth
- discipline
- audit trail
- explicit close conditions
- revalidation and control

He does not want:
- soft state changes
- hidden evidence rules
- pretty but vague UX
- parallel products that break the spine

## Workflow

1. Name Radu's stage:
   - awareness
   - auth
   - compliance onboarding
   - home/triage
   - resolve
   - cockpit
   - dosar
   - monitoring
   - revalidation
   - specialist depth
2. Ask only these questions:
   - Is the case truth explicit?
   - Is the close rule visible?
   - Is evidence serious and attributable?
   - Does monitoring keep the case alive?
   - Does specialist depth support the spine instead of replacing it?
3. Mark the surface as:
   - `disciplinat pentru Radu`
   - `slab ca rigoare`
   - `rupe controlul`
4. If wrong, rewrite toward:
   - clearer state truth
   - stronger evidence gate
   - explicit review/reopen logic
   - better audit visibility

## Guardrails

- Do not optimize away rigor for simplicity.
- Do not accept unresolved ambiguity between states.
- Do not let specialist modules behave like separate products.
- Do not hide monitoring or aftercare.

## Output standard

Return only:
- stage
- what Radu needs here
- rigor gap
- audit/control gap
- pass/fail verdict
