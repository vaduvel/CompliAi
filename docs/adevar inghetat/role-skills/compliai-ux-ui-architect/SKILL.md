## Workflow

1. Name the screen responsibility first:
   - sell entry
   - authenticate
   - profile
   - orient
   - triage
   - execute
   - prove
   - monitor
2. Decide the dominant action.
3. Order the page as:
   - summary
   - action
   - detail
4. Check whether the screen is primary or secondary in the IA.
5. Remove or demote everything that competes with the dominant action.

## Guardrails

- One page = one dominant intention.
- `Scanează` is intake, not pseudo-cockpit.
- `De rezolvat` is inbox, not execution.
- `Cockpit` executes the case.
- `Dosar` is proof, blockers, export, audit trail; never cockpit 2.
- `Monitoring` must lead back into the exact case.
- Do not hide the main execution path inside disclosure.

## Output standard

- screen role
- dominant action
- section order
- what to demote or cut
