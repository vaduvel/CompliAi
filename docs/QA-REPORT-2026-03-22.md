## BUGS CRITICE

### BUG-001 — EROFS crash pe Vercel (filesystem read-only)

| Camp | Valoare |
|------|---------|
| **Endpoint-uri** | `/api/auth/login`, `/api/auth/register`, toate write-urile de state |
| **Eroare** | 500 Internal Server Error — `EROFS: read-only file system` |
| **Cauza** | Aplicatia scria in `.data/` care e read-only pe Vercel serverless |
| **Fix** | `writeFileSafe()` + `appendFileSafe()` — catch EROFS/EACCES gracefully |
| **Commit** | `e6be8ba` |
| **Status** | FIXAT |

### BUG-002 — Inregistrare idempotenta — duplicate Supabase Auth + slug conflict

| Camp | Valoare |
|------|---------|
| **Endpoint** | `POST /api/auth/register` |
| **Eroare** | 500 la retry dupa esec partial — user exista deja in Supabase Auth |
| **Cauza** | `registerSupabaseIdentity()` crasha pe duplicate; `upsertOrganizations()` crasha pe slug unic |
| **Fix** | Lookup user existent in loc de throw; retry cu slug sufixat la conflict |
| **Commit** | `464068a` |
| **Status** | FIXAT |

### BUG-003 — Alert preferences 500 pe Vercel (tabel Supabase inexistent)

| Camp | Valoare |
|------|---------|
| **Endpoint-uri** | `GET /api/alerts/preferences`, `POST /api/alerts/preferences`, `POST /api/alerts/notify`, toate cron-urile digest |
| **Eroare** | 500 — `SUPABASE_STATE_REQUIRED` |
| **Cauza** | Tabelul `alert_preferences` nu exista in Supabase. `CloudPrimaryStorage` esueaza pe query, iar `COMPLISCAN_ALLOW_LOCAL_FALLBACK=false` blocheaza fallback-ul local |
| **Impact cascada** | Intreg pipeline-ul de notificari blocat: prefs → notify → weekly-digest → email delivery |
| **Fix** | 1) Creat tabelul `alert_preferences` in Supabase (`org_id text PK, state jsonb`) 2) Adaugat try/catch rezilent in `readAlertPreferences` + `writeAlertPreferences` |
| **Status** | FIXAT (tabel creat + cod rezilent) |
