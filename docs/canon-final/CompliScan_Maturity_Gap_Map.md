# CompliScan — Maturity Gap Map

**Data:** 2026-03-25  
**Scop:** sa vedem clar, pe domenii, ce exista deja, ce lipseste si ce duce fiecare nucleu la production-grade / 100% maturitate.

**Scara de maturitate:**
- **L1** = concept / directie
- **L2** = exista piese importante, dar fluxul este incomplet
- **L3** = flux utilizabil, dar cu gap-uri de dovada / UX / explainability
- **L4** = aproape production-grade
- **L5** = matur 100%, cap-coada, auditabil, recurenta stabila

---

# 1. GDPR documents + DPA + rights + evidence

## Nivel actual estimat
**L3.5 - L4**

## Ce exista deja
- document generation direction buna
- finding -> document -> dovada
- DPA direction
- evidence / vault direction
- confirmare umana definita serios
- rights / DSAR deja clar prezente in strategie si backlog

## Ce lipseste
- regula universala de confirmare in toate flow-urile
- DSAR complet operational cap-coada
- breach flow ANSPDCP matur
- retention / deletion logic clara
- vendor fingerprinting puternic
- dovada acceptata standardizata pe tipuri
- legal-reviewed final wording pe materialele sensibile

## Ce il duce la L5
- rights / DSAR complet
- ANSPDCP breach path
- evidence acceptance model
- stale data / revalidation
- cookie/site scan legat direct de privacy
- templates validate legal

# 14. Concluzie

Cand am spus ca nucleele acestea sunt cele mai puternice, am vrut sa spun:
- au deja sens de piata
- au deja mai multe piese legate intre ele
- au deja potential de rezolvare, dovada si retention

Dar nu sunt 100% mature pana nu pot:
**detecta -> explica -> ajuta sa rezolvi -> cere dovada corecta -> reverifica -> revalida -> exporta -> rezista la audit**

Asta este definitia reala a maturitatii pentru CompliScan.
