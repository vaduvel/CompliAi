# CompliScan — Screen-by-Screen Blueprint

**Data:** 2026-03-25  
**Scop:** blueprint practic, ecran cu ecran, pentru experienta principala CompliScan.  
**Format:**  
- obiectivul ecranului  
- ce trebuie sa inteleaga userul  
- structura UI  
- copy recomandat  
- CTA principal / CTA secundar  
- ce nu trebuie afisat  

---

# 1. Landing Page

## Obiectiv
Sa explice in 5 secunde:
- ce face CompliScan
- de unde porneste
- ce rezultat concret livreaza

## Ce trebuie sa inteleaga userul
- nu trebuie sa stie el toata legislatia
- aplicatia porneste din date reale
- nu doar scaneaza, ci si ajuta la rezolvare
- tot ce rezolva ramane la dosar

## Structura UI

### Hero
**Headline:**  
Afli ce ti se aplica, rezolvi ce lipseste si pastrezi dovada.

**Subheadline:**  
Pornim din CUI, website si semnalele tale operationale. Apoi CompliScan iti arata ce ti se aplica, ce lipseste, te ajuta sa generezi ce ai nevoie si iti pastreaza istoricul la dosar.

**CTA principal:**  
Incepe gratuit

**CTA secundar:**  
Vezi cum functioneaza

**Subtext sub CTA:**  
Primele rezultate apar in cateva minute. Nu trebuie sa stii legislatia dinainte.

### Bloc 2 — Cum functioneaza
1. Introduci CUI-ul si website-ul  
2. CompliScan verifica ce se poate automat  
3. Vezi ce ti se aplica si ce ai de rezolvat  
4. Generezi ce lipseste si salvezi dovada  
5. Ramai monitorizat in timp

### Bloc 3 — Ce primesti
- Snapshot initial de conformitate
- Top 3 actiuni prioritizate
- Documente si dovezi salvate
- Audit Pack si exporturi
- Monitorizare pentru schimbari relevante

### Bloc 4 — Pentru cine este
- Pentru administratori de IMM-uri
- Pentru contabili si consultanti
- Pentru echipe interne de compliance

## Ce nu trebuie afisat
- jargon tehnic
- lista completa de module
- 10 framework-uri in hero
- promisiuni vagi fara rezultat clar

**Subtitlu:**

### Campuri

- Folosesti eFactura / SPV? (Da / Nu / Nu stiu)
- Cum folosesti CompliScan?

  - Tin mai multe firme

### Ajutor microcopy

Codul de identificare fiscala al firmei. Ex: 12345678 sau RO12345678

**Sub website:**  
Daca ai site, il putem analiza automat pentru primele semnale de conformitate.

## CTA principal

## CTA secundar

## Ce nu trebuie afisat

# 6. Resolve Screen / Prima problema

## Obiectiv
Sa arate clar ca aplicatia nu doar gaseste problema, ci te duce direct la rezolvare.

## Ce trebuie sa inteleaga userul
- ce este problema
- de ce conteaza
- ce castiga daca o rezolva
- unde apasa acum

## Structura UI

### Zona stanga
- titlul problemei
- descriere scurta
- impact
- dovada acceptata

### Zona dreapta
**Titlu:**  
Acum faci asta

1. Genereaza documentul sau deschide flow-ul potrivit
2. Confirma ca reflecta realitatea firmei
3. Salveaza dovada
4. Valideaza si actualizeaza snapshot-ul

### CTA principal
Genereaza acum

### CTA secundar
Am deja dovada

### CTA tertiar
Continui mai tarziu

## Copy recomandat
**Titlu:**  
Politica de confidentialitate lipsa

**Descriere:**  
Fara o politica potrivita, nu poti demonstra usor ce date colectezi si cum informezi persoanele vizate.

**Beneficiu:**  
Dupa acest pas, documentul este salvat la dosar si poate fi folosit ca dovada.

## Ce nu trebuie afisat
- instructiune vaga „mergi in generator”
- lipsa link-ului direct
- confirmare automata fara user

# 14. Formula finala a UI-ului

Pe fiecare ecran, UI-ul trebuie sa raspunda clar la 3 intrebari:

1. Ce vad acum?
2. Ce a facut sistemul pentru mine?
3. Ce fac eu mai departe?

Daca oricare dintre cele 3 lipseste, produsul pare fragmentat.
Daca toate 3 sunt clare, produsul pare inteligent si coerent.
